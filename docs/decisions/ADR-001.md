# ADR-001: Modular Monorepo Foundation

Date: 2026-06-04
Status: Accepted

## Context

RAZON needs backend, frontend, mobile, infrastructure, storage, and
documentation areas that can evolve independently without losing architectural
clarity.

## Decision

Use a monorepo layout with explicit top-level domains:

- `backend`
- `frontend`
- `mobile`
- `infrastructure`
- `storage`
- `docs`

Existing application folders remain untouched until a later migration is
planned.

## Consequences

The project can prepare the target architecture without breaking current code.
Future phases can migrate or connect modules deliberately.
